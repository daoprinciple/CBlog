在Fedora底下使用虛擬機(Virtual Machine)

2017-07-27 | 2019-04-26 | computer |
| 5.4k

最初選擇使用VM的初衷，非是為了要作什麼偉大的事業，而是想讓MS Office可以跑起來。而爲什麼不選擇使用LibreOffice?老實說我也想，但功能上實在是差上一截。如果說能把播放影片這件事情搞定，還有使用pptx的檔案能跟微軟相容，我想我也會選擇使用LibreOffice(爲毛不是MS用開源的軟體格式拉！齁)。也有考慮過用wine安裝Office，但根本裝不起來…。最後選擇用VM裝Win10再加上萬惡的Office…。

¶Update: Using Oracle VM instead of Gnome-Boxes

At the beginning, I just try to use Boxes for Windows. The fatal drawback is that it may crash with unknown reason that I cannot directly figure out. For example, sometimes on my laptop, curses or the keyboard cannot funtion well, they cannot work at the sametime! Moreover, files transmittion is worse! I don’t know exactly the techniques using in Boxes and VM, but VM can load the system immediatelly! Therefore, I do not have to install Dropbox again in Windows.

¶Virtual Machine可以幹嘛？

使用VM就是將作業系統內建另外一作業系統。[1]不需要在兩個作業系統之間作切換。例如，我想要跑Linux的時候順便做Talk，就可以用VM跑兩個環境。當然大神們也許會用VM來測試開發的軟體在不同作業系統、不同的狀態下的相容狀況，這就比較可惜點因為我不需用到這個功能。相對的，他也很大的缺點，不僅會吃很多資源，也無法安裝額外的驅動程式(如果有的話我也想要安裝阿…不然實在有點可惜電腦的強大功能。

¶Oracle VM VirtualBox[2] vs. Gnome-Boxes

一開始想說很多人都在推用VirtualBox，想說也來試試看，結果沒想到真的可以安裝成功！運氣真得不錯。而且使用方式也簡單，基本上環境設定相當容易、檔案傳輸也沒有甚麼問題，唯一的缺點算是吃的資源較多，並且反應速度較為緩慢（有可能機器本身院轉的速度問題。相較之下，嘗試使用Gnome-Boxes。目前使用結果還算滿意，使用就跟一般的原生作業系統幾乎一樣。但檔案傳輸方面的功能卻有待加強，速度不是普通慢。或許還需要一段時間的觀察才能確定自己要用哪一種虛擬機軟體作為日常使用。目前服役中的還是傾向使用Gnome-Boxes，畢竟是開源嘛，多多支持一下。

¶Docker技術[3]

也有人說可以試試看用Docker來跑單獨的程式。但後來發現，Docker的運作是建立在作業系統的原生核心中。換句話說，如果用Windows，那就只會有Window核心（但我看好像還是可以跑Linux阿…黑人問號）[4]，但在Docker在Linux底下卻無法額外安裝Windows的核心跑…我也是醉了。除非說，要安裝Win10，然後再使用Win10作為日常使用軟體，有需要再用Docker跑Linux核心…但這違背使用開源的初衷阿，而且Gnome真的漂亮好看方面易用（推坑）。還有各種調校的個人自訂，想改成啥樣子就可以改成啥樣子。
