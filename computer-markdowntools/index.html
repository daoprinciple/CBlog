電腦筆記 - Markdown相關工具

2018-01-23 | 2018-01-26 | computer |
| 3.6k

現在各種文書處理工具橫行（其實我以前就只知道M$，但開始使用Fedora之後，才知道原來資訊處理的世界如此廣大）一個不小心就入坑了。而這個猛猛的Markdown主攻是簡單容易撰寫，各種基本的文字設定都可以直接寫出來就好。之後有相關的內容就會慢慢增補。

¶基本介紹

如果要看Markdown在此網站上的測試，可以參考另外一篇文章Blog顯示語法測試。

¶文字編輯器

  1. gedit
    這是fedora/gnome內建的文字編輯器，很輕便、優雅。有各種自訂功能可以使用

  2. atom
    超級肥嘟嘟。以前覺得可以加好多套件很帥，但他真的肥到不行。開的很慢，而且本身也佔據容量（雖然現在硬碟不用錢）。個人不太喜愛用這個就是了。

  3. VIM
    開山元祖級的編輯器，不用多說，會想用這個的基本上是DIY狂，已棄坑XD。

  4. Typora <- 極度推薦
    縱然這世界上的MD編輯器多的跟山一樣，但這款編輯器是我遇到最好的一款。寫作及所得。
    有興趣的話可以看他的官方網站與製作理念有需要的朋友可以拿去玩玩，或者喂狗看看他人寫的評論。我要入坑了XD。

Ubuntu 六款 Markdown 編輯器快速試用心得

¶線上編輯器

線上的好處是可以 即時編輯與瀏覽，並且會附上各種文字表示法的小工具，是懶人跟入門寫作的好幫手。一開始在用markdown寫作時用過兩款線上編輯，分別是Dillinger跟Stackedit。Stackedit好處是可以讓你愛愛上雲端隨意的連上各大免空(google drive, dropbox)等等，但同時也是個壞處，就是到處都有你的資料，哈哈。另外一款就比較陽春，但還算是堪用。但我認為這些工具都只是花俏與方便，有時候過度的複雜反而會造成目標的不明確。並且，Markdown的好處就是為了降低編輯門檻，用些簡單的語法渲染處理過後的文字。雖然簡易，但作為一般文書用途在網路上發佈已經足夠。因此，若要成為一名好的文字編輯者，應該使用最基本的文字編輯器作為寫作之用。若真的需要渲染，可以參考本文的其他介紹。

¶轉檔工具

想當然爾，如果想要一稿多投（發佈在各種平臺上），那就必須要靠其他小工具來使用。而**Pandoc**是個很棒的選擇。Pandoc可以用來做各種文檔發佈的轉換工具。除了可以讓Markdown轉成PDF以外，還可以用來轉成電子書epub、HTML等等的格式。而我當初選用Hexo作為文章寫作練習與筆記平臺，是因爲他的擴充良好，而且本身可以用markdown作為寫作語言，已經內建不需要將Markdown轉HTML了，是一款不可多得的好工具阿（推坑）

¶Markdown用Pandoc轉中文Pdf

基本語法

pandoc infile.md -o outfile.pdf

指定Latex的引擎(Xelatex可以渲染中文utf-8編碼。)

pandoc -f markdown_github my.md -o test.pdf --latex-engine=xelatex

如果他說少了xx沒有安裝的話那就裝上去吧，之前再用的時候發現少了mathspec，所以就。

dnf search mathspec
sudo dnf install texlive-mathspec

中文支援換行(自己做一個template…怎麼感覺越來越累人了QAQ)

 pandoc computer-markdowntools.md -o test.pdf --latex-engine=xelatex -V mainfont="SourceHanSansTW-Normal"  --template=template-markdownpdf.tex

Template位置
/usr/share/pandoc-1.19.1/data/templates/

有需要的話記得裝這個

sudo dnf install texlive-xecjk
