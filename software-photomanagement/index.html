影像處理軟體@Linux(Digikam)

2018-02-25 | computer |
| 2.6k

會寫這篇主因爲找尋除了Photoshop以外的替代影像處理軟體。而且，我用的是Fedora 的作業系統，其附加的相片管理程式爲Shotwell，爲Gnome 原生的軟體。但另外一款軟體，DigiKam，則是屬於KDE的桌面系統軟體。究竟這兩個最主要的差異在哪邊呢？而該選擇哪一款軟體作為相片處理用呢？其實，青菜蘿蔔各有所好，兩款軟體各有其專屬用途，並沒有哪一款比較好，哪一款比較不好，只有適合使用的軟體。而以下爲個人使用DigiKam的筆記，以防止以後忘記DigiKam有哪些方便的功能，並且寫下與Shotwell相異的部分。

¶影像管理功能

相片管理程式最重要的便是方便易用，並且可以修圖（這邊的修圖除了美肌等功能外，還有最基本的相片像差修正等的處理）。個人認為，一個好的相片管理軟體必須要包含以下幾個功能：

  1. 相片基本處理，修圖功能精簡且經典

  2. 讀取RAW檔

  3. 影像管理容易（最好排列方式是跟資料夾相同，請養成直接用資料夾管理個人資料的習慣XD）

  4. 越直覺式操作越好（懶人用法），但前提是必須要瞭解每個自動美化的功能究竟是什麼。

    其他的暫時還沒有想到。

¶花樣很多的Digikam

¶批次修改相片

如果說，整組照片都有相同的問題需要修改，又不想要一個一個做的時候，Digikam提供一個簡潔方便的方式，利用類似巨集的方式，將相片修改的步驟分割好幾個（其實就是類似寫程式拉！一個Workflow的概念），的確是滿好使用的。

¶媲美Photoshop的各種修圖功能

Digikam的修圖功能基本上可以媲美Photoshop，當然他不像是Photoshop如此的專業強大（繪圖的話請右轉GIMP）。在Linux系統上的影像處理軟體跟相片管理軟體是分開的（其實我不太瞭解這方面的東西，不過我想Windows底下也有類似的意思，繪圖歸繪圖、照片管理歸照片管理）。在Digikam當中，有一些簡便已設定好的濾鏡可以直接套用（我想現在手機的許多修圖功能都已經比電腦所有的還要多了…）。此外，Digikam有趣的地方在於他將畫面分為基本相片管理（一開始點進去之後的圖）、Image Editor（影像修改）、BatchQueue Manager（批次影像修改）。

¶個人基本的Work Flow

  1. Auto-correction

    基本上每張照片我都會改這個。除非拍照真的很糟糕或者是想要改一些有趣的東西，否則基本上不需要手動調整色溫、亮度等（其實是懶惰拉…我沒有要發表所以就不搞這些東西）

  2. Lens-correction 鏡頭影像修正（選擇性）

    這其實有點討厭，用手動一張一張改的話，軟體可以讀取到資訊，但是還是必須要手動選取鏡頭（可能是鏡頭太舊了沒有支援，只能找相似的作為修正）。使用結果認為十分良好。

  3. 最後就是選擇照片格式輸出拉！（也可以不要）

    這看每個人的選擇，個人選擇jpg檔案做爲最後發佈的影像。

  4. 上傳至雲端並分享

¶另類選擇 -基本精巧的Shotwell

Shotwell，在Gnome所開發的軟體，因此，其給予的精神是極簡化的應用程式（不需要作太多額外的功能設定，也不太會有太多需要自己設定的東西，最主要是簡單、直觀地使用）。基於這樣的理由，Shotwell提供的功能只有基本的編輯，比較進階的相片編輯功能Shotwell基本上都沒有。但是他也是有優點，如果只是要做基本的相片整理、上傳、亮度調整、影像剪裁，使用Shotwell肯定是夠用的。
